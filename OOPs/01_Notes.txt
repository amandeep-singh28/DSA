->Constructor Working in Inheritance
(i)Java internally called parent class everytime even if we don't write super().
(ii)If there are only default constructor present in the both the classes, than first parent class default constructor will run and than child class default constructor.
(iii)super() must be the first statement inside the default constructor of the child class.
(iv)super() must not be written twice.
(v)If the parent class is having parameterized constructor than we need to specify super() in the child class constructor and also in the parameters inside the super() keyword
(vi)In case of default constructor super() is automatically called.

Example:-

class Student {
    Student() { // Default Constructor
        System.out.println("Parent class");
    }
}

class Hostel extends Student {
    Hostel() { // Default Constructor
        // super()
        System.out.println("Child class");
    }
}

public class constructorWorking {
    public static void main(String[] args) {
        Hostel h1 = new Hostel();
    }
}

Example:-

package OOPs;

class Student {
    Student(int x) {
        System.out.println("Parent class");
    }
}

class Hostel extends Student {
    Hostel(String a) {
        super(10);
        System.out.println("Child class");
    }
}

public class constructorWorking {
    public static void main(String[] args) {
        Hostel h1 = new Hostel("Hello");       
    }
}


->Method overriding and Method overloading
(i)Method overriding
-Same function name and same parameters.
-Cannot be done in the same class.
-It only happens when class inherits something(Inheritance).

(ii)Method overloading
-Same function name and different parameters.
-Can be done in the same class.


->Uses of super()
(i)It can access the overriden methods
Example:-

class ABC {
    void display() {
        System.out.println("ABC class");
    }
}

class XYZ extends ABC {
    void display() {
        super.display();
        System.out.println("XYZ class");
    }
}

public class useOfSuper {
    public static void main(String[] args) {
        XYZ obj1 = new XYZ();
        obj1.display();
    }
}

(ii)It can access public data members of the parent class

Example:-

class ABC {
    String name = "Amandeep Singh";
    void display() {
        System.out.println("ABC class");
    }
}

class XYZ extends ABC {
    String name = "Amandeep";
    void display() {
        super.display();
        System.out.println(name);
        System.out.println(super.name);
    }
}

public class useOfSuper {
    public static void main(String[] args) {
        XYZ obj1 = new XYZ();
        obj1.display();
    }
}

(iii)It can call the default constructor of the parent class.
-super() must be the first statement inside the child class default constructor

Example:-

class ABC {
    ABC() {
        System.out.println("ABC Class");
    }
}

class XYZ extends ABC {
    XYZ() {
        super();
        System.out.println("XYZ Class");
    }
}

public class useOfSuper {
    public static void main(String[] args) {
        XYZ obj1 = new XYZ();
    }
}

(iv)To access the parameterized constructor of the parent class

Example:-

class ABC {
    ABC() {
        System.out.println("ABC Class");
    }
    ABC(String a) {
        System.out.println("Name:" + a);
    }
}

class XYZ extends ABC {
    XYZ() {
        super("Amandeep");
        System.out.println("XYZ Class");
    }
}

public class useOfSuper {
    public static void main(String[] args) {
        XYZ obj1 = new XYZ();
    }
}

->Uses of final keyword

(i)Using final with a variable
When we create a variable with final keyword, we cannot update the value of that variable.

Example:-
public class useOfFinal {
    public static void main(String[] args) {
        final int a = 10;
        System.out.println(a);
        a = 20; // It will throw an error(Compile Time Error)
        System.out.println(a);
    }
}

(ii)Using final along with methods
It prevents overriding(Modification)
A final method can be inherited but cannot be overridden because it prevents modification of behavior in child classes.

Example:-
class ABC {
    final void display() {
        System.out.println("ABC Class");
    }
}

class XYZ extends ABC {
    // void display() { // This will throw COMPILATION ERROR
    //     System.out.println("XYZ Class");
    // }
}
public class useOfFinal {
    public static void main(String[] args) {
        XYZ obj1 = new XYZ(); // But the method can be inherited in the base class
        obj1.display();
    }
}

(iii)Using final along with class
Class will not be inheritable

Example:-
final class ABC { // Not inheritable
}

class PQR {

}
class XYZ extends ABC { // Throw error

}
class RST extends PQR {

}

(iv)Using final along with static
static means it belongs to the class not to the object.
static = single shared value

-Without static
class Student {
    String name = "Amandeep";
}

public class Main {
    public static void main(String[] args) {
        Student s1 = new Student();  // object needed (Copies -> Many)
        System.out.println(s1.name);
    }
}
-With static
class Student {
    static String college = "LPU";
}

public class Main {
    public static void main(String[] args) {
        System.out.println(Student.college); // no object needed (Copy -> Only One)
    }
}


-Now final + static
static → belongs to class, one shared copy
final → cannot be changed

Example:-

class ABC {
    final static int a = 10;
}
public class useOfFinal {
    public static void main(String[] args) {
        // ABC.a = 20;  // Cannot be changed
        System.out.println(ABC.a);
    }
}

****Types of Inheritance in Java****
-Java supports single, multilevel, and hierarchical inheritance with classes, while multiple and hybrid inheritance are achieved using interfaces.

(i)Single level Inheritance
One parent one child

Example:-
class A {

}
class B extends A {

}

(ii)MultiLevel Inheritance (Chain of inheritance)
A class inherits from another class, and then a third class inherits from that child.

Example:-
class A {

}
class B extends A { 
    
}

class C extends B { // Multi level Inheritance

}

(iii)Hierarchical Inheritance
One parent and n childrens

class A {

}
class B extends A { 
    
}

class C extends A {

}

class D extends A { // Hierarchical Inheritance

}
